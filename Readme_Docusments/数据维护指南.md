# 📋 末日生存游戏 — 数据维护指南

## 核心原则

- **所有游戏数据集中在 `js/config.js`**
- 修改 `config.js` 后刷新浏览器即可生效
- 其他文件（`combat.js`、`maps.js`、`state.js` 等）不需要修改
- 新增物品只需在对应数组中添加条目
- `config.js` 内部有些常量是自动派生的（如 `NURSE_MEDICINE_POOL`、`V_TRADE_AMMO_TYPES`、`BUILDING_MATERIAL_NAMES`），修改源数据后自动更新

## 二、常见操作指南

### 2.1 新增食物

在 `FOODS` 数组末尾添加条目即可。

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 唯一标识，全局唯一 |
| `name` | string | ✅ | 显示名称 |
| `type` | string | ✅ | 固定为 `"food"` |
| `hunger` | number | ✅ | 恢复饱腹值 |
| `hydration` | number | ❌ | 恢复水分值，省略则不恢复水分 |

**示例：添加"红烧肉"**

```js
// 在 FOODS 数组末尾添加
{ id: "红烧肉", name: "红烧肉", type: "food", hunger: 50, hydration: 10 },
```

添加后，作弊系统自动支持 `/get_food_红烧肉_10` 格式获取。地图掉落中 `food` 类型会自动包含新食物。

---

### 2.2 新增近战武器

在 `MELEE_WEAPONS` 数组末尾添加条目。

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 唯一标识 |
| `name` | string | ✅ | 显示名称 |
| `type` | string | ✅ | 固定为 `"melee"` |
| `damage` | number | ✅ | 基础伤害值 |
| `durability` | number | ✅ | 最大耐久度（拳头为 `Infinity`） |
| `rarity` | string | ❌ | 稀有度：`common`/`uncommon`/`rare`/`epic`/`legendary`，拳头无此字段 |
| `comboRate` | number | ✅ | 连击概率（0~1） |

**示例：添加"链锯剑"**

```js
{ id: "链锯剑", name: "链锯剑", type: "melee", damage: 110, durability: 90, rarity: "epic", comboRate: 0.16 },
```

**稀有度与连击率对应关系：**

| 稀有度 | 连击率 | 伤害范围参考 |
|--------|--------|-------------|
| common | 0.05 | 30~48 |
| uncommon | 0.08 | 40~70 |
| rare | 0.12 | 66~90 |
| epic | 0.15 | 99~105 |
| legendary | 0.18 | 118+ |

---

### 2.3 新增远程武器

添加远程武器需要**同时关注弹药关联**，步骤如下：

**步骤1：在 `RANGED_WEAPONS` 添加武器**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 唯一标识 |
| `name` | string | ✅ | 显示名称 |
| `type` | string | ✅ | 固定为 `"ranged"` |
| `damage` | number | ✅ | 基础伤害值 |
| `integrity` | number | ✅ | 完整度（通常为100） |
| `ammoType` | string | ✅ | 使用的弹药类型，必须与 `AMMO` 中的弹药 `id` 对应 |
| `critRate` | number | ✅ | 暴击概率（0~1） |
| `rarity` | string | ✅ | 稀有度 |

**步骤2：如果需要新弹药类型，在 `AMMO` 添加弹药**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 弹药唯一标识 |
| `name` | string | ✅ | 显示名称 |
| `type` | string | ✅ | 固定为 `"ammo"` |
| `compatibleWith` | string[] | ✅ | 兼容武器ID列表 |

**步骤3：如果使用已有弹药，在对应弹药的 `compatibleWith` 数组中添加新武器ID**

**示例：添加"SCAR-H"（使用已有弹药 7.62×51mm）**

```js
// 1. 在 RANGED_WEAPONS 末尾添加
{ id: "SCAR-H", name: "SCAR-H", type: "ranged", damage: 85, integrity: 100, ammoType: "7.62×51mm", critRate: 0.28, rarity: "rare" },

// 2. 在 AMMO 中找到 "7.62×51mm"，在 compatibleWith 数组中添加 "SCAR-H"
{ id: "7.62×51mm", name: "7.62×51mm", type: "ammo", compatibleWith: ["M700", "FAL", "EBR14", "SR25", "SCAR-H"] },
```

**示例：添加"SCAR-H"（使用全新弹药 7.62×51mm NATO）**

```js
// 1. 在 RANGED_WEAPONS 末尾添加
{ id: "SCAR-H", name: "SCAR-H", type: "ranged", damage: 85, integrity: 100, ammoType: "7.62×51mm NATO", critRate: 0.28, rarity: "rare" },

// 2. 在 AMMO 末尾添加新弹药
{ id: "7.62×51mm NATO", name: "7.62×51mm NATO", type: "ammo", compatibleWith: ["SCAR-H"] },
```

**暴击率参考（按武器类别）：**

| 武器类别 | 暴击率范围 | 弹药类型 |
|----------|-----------|---------|
| 弓/弩 | 0 | 箭矢 |
| 手枪 | 0.15~0.20 | 9×19mm / .357 Magnum |
| 冲锋枪 | 0.18~0.23 | 9×19mm |
| 步枪(5.56) | 0.22~0.25 | 5.56×45mm NATO |
| 步枪(7.62×39) | 0.20~0.28 | 7.62×39mm |
| 战斗步枪(7.62×51) | 0.22~0.32 | 7.62×51mm |
| 狙击枪 | 0.35~0.45 | 7.62×51mm / .300 Win Mag |
| 霰弹枪 | 0.10~0.12 | 12号霰弹 |

### 2.4 修改BOSS掉落

在 `FIXED_LOOT_DROPS` 中修改对应条目。

**当前固定掉落一览：**

| 键名 | 来源 | 当前掉落 |
|------|------|---------|
| `banker_kill` | 击杀银行行长 | 武士刀（近战） |
| `leader_gift` | 阵地首领赠送 | 唐横刀（近战） |
| `doctor_trade` | 陈博士交易 | M700 + 7.62×51mm×30 |
| `outlaw_kill` | 击杀马三 | AK47 + 7.62×39mm×30 |
| `shadow_kill` | 击杀黑影 | GP100 + .357 Magnum×20 |
| `space_crate` | 航天基地箱子 | AWM + .300 Win Mag×30 |
| `tunnel_cache` | 隧道缓存 | 9×19mm×20 + 压缩饼干 |

**示例：把马三掉落从 AK47 改为 HK416**

```js
outlaw_kill: { weaponId: "HK416", type: "ranged", ammoId: "5.56×45mm NATO", ammoCount: 30 },
```

**示例：把银行行长掉落改为远程武器**

```js
banker_kill: { weaponId: "EBR14", type: "ranged", ammoId: "7.62×51mm", ammoCount: 30 },
```

**注意：** `type` 字段决定从哪个数组查找武器，`"melee"` 从 `MELEE_WEAPONS` 查找，`"ranged"` 从 `RANGED_WEAPONS` 查找。如果掉落包含弹药，`ammoId` 必须与 `AMMO` 中的 `id` 对应。

### 2.5 修改黑市商人商品

在 `TRADER_WEAPON_SHOP` 中增删条目。

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `weaponId` | string | 武器ID，必须在 `MELEE_WEAPONS` 或 `RANGED_WEAPONS` 中存在 |
| `type` | string | `"melee"` 或 `"ranged"` |
| `costMin` | number | 最低香烟价格 |
| `costMax` | number | 最高香烟价格 |

**示例：添加新商品"小匕首"**

```js
{ weaponId: "小匕首", type: "melee", costMin: 3, costMax: 5 },
```

**示例：添加"MP5"远程武器**

```js
{ weaponId: "MP5", type: "ranged", costMin: 15, costMax: 20 },
```

### 2.6 修改地图掉落概率

在 `MAPS` 的 `lootTable` 中调整权重。权重是相对值，不需要加起来等于100，但习惯上总和为100方便理解。

**lootTable 可用的 key：**

| key | 说明 | 对应数据源 |
|-----|------|-----------|
| `food` | 食物 | `FOODS` |
| `drink` | 饮品 | `DRINKS` |
| `fruit` | 水果 | `FRUITS` |
| `cigarette` | 香烟 | `CIGARETTES` |
| `medicine` | 药品 | `MEDICINES`（按稀有度加权） |
| `backpack` | 背包 | `LOOT_BACKPACKS`（按稀有度加权） |
| `melee` | 近战武器 | `MELEE_WEAPONS`（排除拳头，按稀有度加权） |
| `ranged` | 远程武器 | `RANGED_WEAPONS`（按稀有度加权） |
| `ammo` | 弹药 | `AMMO`（低危地图箭矢概率高） |
| `building` | 建材 | `BUILDING_MATERIALS` |
| `empty` | 无掉落 | — |

**示例：增加"废弃警察局"的远程武器掉落率**

```js
// 原始
{ id: "废弃警察局", name: "废弃警察局", danger: "★★★★高危", encounterRate: 0.55,
  lootTable: { melee: 15, ranged: 15, ammo: 25, medicine: 10, backpack: 10, fruit: 10, food: 5, drink: 5, empty: 5 } },

// 修改后：提高远程武器权重到25，降低弹药到15
{ id: "废弃警察局", name: "废弃警察局", danger: "★★★★高危", encounterRate: 0.55,
  lootTable: { melee: 15, ranged: 25, ammo: 15, medicine: 10, backpack: 10, fruit: 10, food: 5, drink: 5, empty: 5 } },
```

### 2.8 新增地图

在 `MAPS` 数组中添加新条目。

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 唯一标识，同时用于丧尸池分配 |
| `name` | string | ✅ | 显示名称 |
| `danger` | string | ✅ | 危险等级显示文本 |
| `encounterRate` | number | ✅ | 遭遇率（0~1），0表示不会遭遇丧尸 |
| `noZombie` | boolean | ❌ | 设为 `true` 则完全没有丧尸 |
| `lootTable` | object | ✅ | 掉落权重表 |

**危险等级参考：**

| 等级 | `danger` 值 | `encounterRate` 参考 |
|------|------------|---------------------|
| 安全 | `"☆安全"` | 0 |
| 轻松 | `"★轻松"` | 0.03~0.10 |
| 低危 | `"★★低危"` | 0.12~0.20 |
| 中危 | `"★★★中危"` | 0.35~0.40 |
| 高危 | `"★★★★高危"` | 0.50~0.55 |
| 绝境 | `"★★★★★绝境"` | 0.60~0.70 |
| 炼狱 | `"★★★★★★炼狱"` | 0.80~0.90 |

**示例：添加"废弃地铁站"**

```js
{
  id: "废弃地铁站",
  name: "废弃地铁站",
  danger: "★★★★高危",
  encounterRate: 0.5,
  lootTable: { ammo: 20, ranged: 10, melee: 10, food: 15, drink: 15, medicine: 10, fruit: 10, backpack: 5, empty: 5 }
},
```

**⚠️ 重要提醒：** 新增地图后，还需要在 `getRandomZombie` 函数的 `poolMap` 中添加地图ID到丧尸池的映射，否则新地图会默认使用低危丧尸池：

```js
const poolMap = {
  // ... 已有映射 ...
  "废弃地铁站": highPool,  // 添加这一行
};
```

### 2.9 修改NPC任务

在 `SURVIVOR_NPC` 的 `quests` 对象中修改。

**任务字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 任务名称 |
| `desc` | string | 任务描述 |
| `reqAffection` | number | 所需好感度 |
| `require` | object | 提交需求 |
| `reward` | object | 奖励 |
| `story` | string | 任务剧情文本 |

**require 支持的格式：**

```js
// 食物+饮品
require: { food: 1, drinks: 1 }

// 指定药品
require: { medicine: 5, medicineId: "止血带" }

// 指定饮品
require: { drinks: 10, drinkId: "纯牛奶" }

// 多种物品组合
require: { items: [{ type: "medicine", id: "医用急救包", count: 6 }, { type: "medicine", id: "抗感染血清", count: 6 }] }

// 所有药品各N个
require: { allMedicine: 2 }

// 所有罐头各1个
require: { allCanned: 1 }

// 所有水果各1个
require: { allFruits: 1 }

// 所有饮品+所有食物各1个
require: { allDrinks: 1, allFoods: 1 }
```

**reward 支持的格式：**

```js
// 单个物品
reward: { item: "小匕首", desc: "小匕首×1" }

// 物品+弹药
reward: { item: "P226", ammo: { type: "9×19mm", count: 50 }, desc: "P226×1 + 9×19mm×50" }

// 多个物品
reward: { itemStack: [{ id: "手术包", count: 3 }], desc: "手术包×3" }

// 香烟+弹药
reward: { cigarettes: 20, ammo: { type: "9×19mm", count: 50 }, desc: "随机香烟×20 + 9×19mm×50" }
```

```js
v1: { name: "初见任务", desc: "上交1份食物+1份饮品", reqAffection: 0, require: { food: 1, drinks: 1 }, reward: { item: "大扳手", desc: "大扳手×1" }, story: "\"饿着肚子没法战斗。给我吃的，我给你工具。\" 她用扳手敲了敲墙壁。" },
```

## 三、稀有度体系说明

游戏使用5级稀有度体系，影响掉落概率和武器属性：

### 稀有度等级

| 稀有度 | 英文 | 中文 | 掉落权重 | 颜色参考 |
|--------|------|------|---------|---------|
| common | common | 普通 | 20 | 白/灰 |
| uncommon | uncommon | 优秀 | 12 | 绿 |
| rare | rare | 稀有 | 6 | 蓝 |
| epic | epic | 史诗 | 3 | 紫 |
| legendary | legendary | 传奇 | 1 | 橙/金 |

### 掉落权重说明

掉落权重用于 `pickRandomLoot` 函数中的加权随机选取。权重越高，出现概率越大。实际概率计算方式：

```
某稀有度概率 = 该稀有度权重 / 所有稀有度权重总和
```

以近战武器为例（权重总和 = 20+12+6+3+1 = 42）：

| 稀有度 | 权重 | 大致概率 |
|--------|------|---------|
| common | 20 | 47.6% |
| uncommon | 12 | 28.6% |
| rare | 6 | 14.3% |
| epic | 3 | 7.1% |
| legendary | 1 | 2.4% |

### 近战武器连击率与稀有度对应关系

| 稀有度 | 连击率 | 现有武器示例 |
|--------|--------|-------------|
| （拳头） | 0 | 拳头 |
| common | 0.05 | 平底锅、大扳手、棒球棍、铁管、铁铲 |
| uncommon | 0.08 | 小匕首、屠刀、警棍、撬棍、羊角锤、消防斧、军工刀 |
| rare | 0.12 | 指虎刀、大砍刀、西洋剑、龙抄剑、长骑枪 |
| epic | 0.15 | 武士刀、唐横刀、电锯 |
| legendary | 0.18 | 寰宇之剑 |

### 远程武器暴击率范围

## 四、数据关联图

以下常量之间存在引用关系，修改时需注意联动：

### 弹药 ↔ 武器（双向关联）

```
AMMO.compatibleWith[]  ←→  RANGED_WEAPONS.ammoType
```

- 每把远程武器的 `ammoType` 必须在 `AMMO` 中有对应条目
- 每种弹药的 `compatibleWith` 必须包含所有使用该弹药的武器ID
- **新增远程武器时**：如果使用已有弹药，需在弹药的 `compatibleWith` 中添加武器ID
- **新增弹药时**：需确保所有使用该弹药的武器都在 `compatibleWith` 中

### 固定掉落 → 物品数组

```
FIXED_LOOT_DROPS.weaponId  →  MELEE_WEAPONS / RANGED_WEAPONS
FIXED_LOOT_DROPS.ammoId    →  AMMO
FIXED_LOOT_DROPS.foodId    →  FOODS
```

- `type` 字段为 `"melee"` 时，`weaponId` 从 `MELEE_WEAPONS` 查找
- `type` 字段为 `"ranged"` 时，`weaponId` 从 `RANGED_WEAPONS` 查找

### 默认物品 → 各物品数组

```
DEFAULT_ITEM_IDS.melee  →  MELEE_WEAPONS
DEFAULT_ITEM_IDS.food   →  FOODS
DEFAULT_ITEM_IDS.drink  →  DRINKS
DEFAULT_ITEM_IDS.seed   →  BUILDING_MATERIALS
DEFAULT_ITEM_IDS.serum  →  MEDICINES
```

### 自动派生关系

```
NURSE_MEDICINE_POOL  ←  MEDICINES.filter(rarity === "common" || rarity === "uncommon")
V_TRADE_AMMO_TYPES   ←  AMMO.filter(id !== "箭矢").map(id)
BUILDING_MATERIAL_NAMES  ←  BUILDING_MATERIALS 遍历生成 {id: name}
```

修改源数据后，这些派生常量会自动更新，无需手动修改。

### NPC任务奖励 → 物品数组

```
SURVIVOR_NPC.quests.reward.item      →  MELEE_WEAPONS / RANGED_WEAPONS
SURVIVOR_NPC.quests.reward.ammo.type →  AMMO
SURVIVOR_NPC.quests.require.medicineId →  MEDICINES
SURVIVOR_NPC.quests.require.drinkId    →  DRINKS
```

### 交易相关

```
TRADER_WEAPON_SHOP.weaponId  →  MELEE_WEAPONS / RANGED_WEAPONS
LILI_REWARD_MEDICINE_IDS     →  MEDICINES
XIAOHAN_REWARD_FOOD_IDS      →  FOODS
TRADE_TEMPLATES.ammoType     →  AMMO
```

### 地图 → 丧尸池

```
MAPS.id  →  getRandomZombie() 中的 poolMap 映射
```

新增地图ID后需在 `poolMap` 中添加对应映射。

### 罐头列表

```
CANNED_FOOD_IDS  →  FOODS（子集）
```

用于苏小涵的"罐头收集"任务，如果新增罐头类食物，需同步更新此数组。

### 工具武器

```
TOOL_WEAPON_IDS  →  MELEE_WEAPONS（子集）
```

标识哪些近战武器属于"工具"类别。

## 五、注意事项

### 1. id 必须全局唯一

每个物品的 `id` 在其所属数组内必须唯一。建议跨数组也保持唯一，避免混淆。

### 2. 修改 id 会影响存档兼容性

玩家的存档中保存的是物品 `id`，如果修改了某个物品的 `id`，旧存档中引用该 `id` 的数据将无法正确匹配。**如需改名，建议只修改 `name` 字段，保持 `id` 不变。**

### 3. 新增物品不需要修改其他文件

在对应数组中添加条目即可，掉落系统、作弊系统等会自动识别新物品。

### 4. 删除物品需要检查所有引用

删除物品前，必须检查以下位置是否引用了该物品：
- `FIXED_LOOT_DROPS` 中的 `weaponId`、`ammoId`、`foodId`
- `DEFAULT_ITEM_IDS` 中的各字段
- `AMMO` 中的 `compatibleWith`
- `TRADER_WEAPON_SHOP` 中的 `weaponId`
- `SURVIVOR_NPC` 中任务的 `require` 和 `reward`
- `LILI_REWARD_MEDICINE_IDS` 和 `XIAOHAN_REWARD_FOOD_IDS`
- `CANNED_FOOD_IDS`
- `TOOL_WEAPON_IDS`
- `TRADE_TEMPLATES` 中的 `ammoType`

### 5. rarity 字段必须是5个值之一

有效的稀有度值：`"common"`、`"uncommon"`、`"rare"`、`"epic"`、`"legendary"`。如果使用了不在这5个值中的稀有度，掉落系统中的加权随机将使用默认权重1，可能导致非预期行为。

### 6. 弹药 compatibleWith 必须与武器 ammoType 对应

- 每把远程武器的 `ammoType` 必须在 `AMMO` 中存在
- `AMMO` 中每种弹药的 `compatibleWith` 必须包含所有使用该弹药的武器
- 两边必须保持同步，否则会导致弹药无法装填或武器无法射击

### 7. FIXED_LOOT_DROPS 中的 type 决定查找范围

- `type: "melee"` → 从 `MELEE_WEAPONS` 查找 `weaponId`
- `type: "ranged"` → 从 `RANGED_WEAPONS` 查找 `weaponId`
- 如果 `type` 与实际武器类型不匹配，将找不到对应武器

### 8. 地图ID与丧尸池映射

`getRandomZombie` 函数使用地图 `id` 精确匹配丧尸池。新增地图时务必在 `poolMap` 中添加映射，否则默认使用低危丧尸池。

### 9. 作弊命令格式

作弊系统使用物品类型和ID生成命令，格式为 `/get_{type}_{id}_{count}`。其中 `type` 对应物品的 `type` 字段值。例如：
- `/get_food_压缩饼干_5`
- `/get_melee_武士刀_1`
- `/get_ranged_AK47_1`
- `/get_medicine_抗生素_3`
