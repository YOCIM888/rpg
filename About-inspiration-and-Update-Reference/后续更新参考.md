## 写在前面

首先在做游戏这件事情上，我一直是个菜鸟，但是我很喜欢这个项目。
这个项目经历了数十个测试版本的迭代，从最初的一个简单文字冒险，成长为一个拥有85把武器、16种丧尸、12个结局、10+条任务链的完整游戏。

它不完美，但它是完整的。

后续可能不会像之前那样高频更新了——但这份文档会留在这里。无论是一周后、一年后、还是五年后，当你想回来看一眼的时候，这里有你需要的所有信息。

致谢所有参与过这个项目的AI助手们——你们是这段旅程中不可或缺的伙伴，给了我很多帮助和灵感，也是这个项目的成功所在。

## 一、可直接落地的小型内容包

> 这些内容不需要改动核心架构，只需在现有数据文件中添加条目和少量逻辑代码。

### 1.1 丧尸之王叙事补完包

**问题**：丧尸之王"寂灭"是游戏最强常规BOSS，但击败后无特殊剧情，"寂灭"的哲学意味未被利用。

**建议**：
- 击败后掉落【寂灭之心】（特殊物品），触发一段独白文本
- 独白暗示寂灭曾是人类——也许是研究所的第一任所长
- 寂灭之心可交给陈博士，解锁一段关于病毒起源的对话
- 不改变任何结局，只是补完叙事

**涉及文件**：`maps.js`（击败后逻辑）、`story-dialogues.js`（独白文本）、`castle.js`（新增特殊物品）

### 1.2 巨型傀儡来历补完包

**问题**："傀儡"暗示人为制造，但游戏中未展开。

**建议**：
- 航天基地新增"研究日志"道具（搜刮时概率获得，共5页）
- 日志内容揭示：傀儡是末日前军事项目的产物，用于守卫关键设施
- 日志最后一页暗示病毒与军事实验有关
- 集齐5页日志可交给陈博士，解锁额外对话

**涉及文件**：`map-actions.js`（新增搜刮选项）、`story-dialogues.js`（日志文本）、`castle.js`（新增道具）

### 1.3 岛屿NPC任务链包

**问题**：顾月和林寒只有7条对话，无任务链，功能单一。

**建议**：

**顾月任务链（3步）**：
1. "国王的勋章" — 顾月想找到国王年轻时在岛上留下的纪念物（需在岛屿探索中找到）
2. "信仰的考验" — 岛屿遭遇丧尸登陆，顾月质疑国王为何不派援军
3. "自己的光" — 顾月不再盲从，开始为岛民而非国王做事。奖励：岛屿商店折扣

**林寒任务链（3步）**：
1. "毒鱼之谜" — 调查为什么近海鱼群突然带毒（暗示城堡的病毒研究污染了海域）
2. "海底的异响" — 深海传来不明声音，林寒怀疑与城堡实验有关
3. "选择站队" — 林寒要求你表态：忠于国王，还是忠于真相。奖励：毒海鱼处理效率提升

**涉及文件**：`guyue.js`、`linhan.js`（新增任务函数）、`yumo-dialogues.js`（新增对话）、`castle.js`（新增道具）

### 1.4 地图事件扩展包

**问题**：部分地图交互选项较少，探索感不足。

**建议**：
- 废弃警察局：新增"翻找档案室"选项，发现末日前警方的丧尸事件报告
- 地下地铁隧道：新增"隧道深处的广播"选项，听到断断续续的求救信号
- 军事检查站：新增"老赵的过去"对话链，了解末日前军事行动的真相
- 工业园区：新增"流水线上的痕迹"选项，发现末日前生产的不明药剂

**涉及文件**：`map-actions.js`、`map-dialogues.js`

## 二、中型系统扩展

> 这些内容需要新增一些系统逻辑，但不需要改动核心架构。

### 2.1 战斗技能系统

**问题**：战斗系统偏简单，近战/远程二选一，缺乏策略深度。

**建议**：

**近战技能（3个）**：
| 技能 | 效果 | 解锁条件 |
|------|------|----------|
| 猛击 | 单次伤害×1.5，消耗2回合 | 近战击杀30只丧尸 |
| 格挡 | 本次受到伤害减半 | 装备盾牌类武器 |
| 连斩 | 连续攻击2次，每次伤害×0.7 | 近战击杀80只丧尸 |

**远程技能（3个）**：
| 技能 | 效果 | 解锁条件 |
|------|------|----------|
| 精准射击 | 命中率+30%，消耗2回合 | 远程击杀30只丧尸 |
| 压制射击 | 丧尸本回合无法攻击 | 装备冲锋枪/轻机枪 |
| 爆头 | 概率一击必杀（BOSS除外） | 远程击杀80只丧尸 |

**涉及文件**：`combat.js`（核心改动）、`state.js`（新增技能状态）、`ui.js`（技能显示）

### 2.2 装备特效系统

**问题**：武器只有伤害/耐久/弹药等基础属性，缺乏差异化。

**建议**：

| 特效 | 效果 | 适用武器示例 |
|------|------|-------------|
| 燃烧 | 命中后3回合持续伤害 | 燃烧瓶、火焰喷射器 |
| 冰冻 | 命中后概率冻结1回合 | 冰霜弹、冷冻枪 |
| 穿透 | 无视护甲 | 穿甲弹、狙击枪 |
| 击退 | 命中后概率击退 | 霰弹枪 |
| 感染 | 命中后增加感染值 | 毒刃、腐蚀弹 |
| 吸血 | 命中后恢复少量生命 | 吸血鬼之刃（新武器） |

**涉及文件**：`melee.js`/`ranged.js`（新增特效字段）、`combat.js`（特效触发逻辑）

### 2.3 基地建设叙事化

**问题**：基地升级只是数值变化，缺乏叙事反馈。

**建议**：
- 每级基地升级时触发一段剧情文本
- 3级基地：有流浪者敲门求助，可选择收留或拒绝（影响后续事件）
- 4级基地：基地遭遇小型尸潮，需防守（战斗事件）
- 5级基地：基地成为小型社区，NPC会来拜访

**涉及文件**：`base-actions.js`、`base-levels.js`（新增叙事字段）

### 2.4 钓鱼深度扩展

**问题**：钓鱼系统5种稀有度但缺乏特殊鱼种的故事或用途。

**建议**：
- 新增3种传说级鱼种，各有独特用途：
  - **深渊灯笼鱼** — 发光，可替代手电筒在夜间探索
  - **变异巨蟹** — 交给林寒可制作特殊护甲
  - **幽灵水母** — 交给陈博士可提取微量改良血清成分
- 新增"夜钓"机制：深夜钓鱼概率不同，传说鱼种只在夜间出现

**涉及文件**：`fish.js`（新增鱼种）、`fishing.js`（夜钓逻辑）

## 三、大型DLC级扩展

> 这些内容需要新增大量代码和数据，相当于游戏的一个扩展包。

### 3.1 DLC：病毒起源

**主题**：揭示丧尸病毒的真实来源

**新增内容**：
- **地下实验室**（新地图，绝境级）— 位于岛屿地下，余墨公爵任务链5完成后解锁入口
- **研究档案**（新道具系列，共10份）— 散落在各地图，集齐后拼凑出完整真相
- **零号病人**（新BOSS）— 实验室最深处的变异体，曾是研究团队的核心成员
- **真相结局**（第13个结局）— 揭示病毒是军事实验意外泄漏，选择公开真相或隐瞒

**核心剧情**：
```
丧尸病毒源于一项代号"重生"的军事实验
  └→ 目标：让士兵在战场上自我修复伤口
  └→ 副作用：修复失控，细胞无限增殖，意识消亡
  └→ 零号病人是项目首席研究员，主动注射了第一针
  └→ 她变成了第一个丧尸，也是最强的丧尸——比寂灭更古老
  └→ 城堡的国王知道真相（末日前就是他资助的项目）
  └→ 陈博士也知道（他是项目的前成员）
```

**涉及文件**：新增 `js/data/maps/underground-lab.js`、`js/island/underground.js`、修改多个现有文件

### 3.2 DLC：第三条路

**主题**：岛屿脱离城堡，成为独立势力

**新增内容**：
- **岛屿独立路线** — 余墨公爵在灵魂伴侣阶段后，可选择支持他脱离城堡
- **岛屿议会** — 新的政治系统，玩家可参与岛屿决策
- **海战系统** — 城堡派舰队镇压，需防守岛屿
- **自由之岛结局**（第14个结局）— 岛屿成为末世中第一个独立城邦

**核心剧情**：
```
余墨公爵觉醒
  └→ "我守了这座岛这么久，不是为了国王，是为了岛上的人。"
  └→ 玩家帮助余墨建立独立防御
  └→ 城堡派舰队镇压 → 海战防守
  └→ 成功后，岛屿宣布独立
  └→ "这座岛不属于国王，不属于任何人——它属于岛上的每一个人。"
```

### 3.3 DLC：新周目+

**主题**：多周目继承系统，降低重复体验

**新增内容**：
- **周目继承**：新周目继承已解锁结局数量对应的加成
  - 3个结局：初始物资+50%
  - 6个结局：初始武器升级
  - 9个结局：解锁"速通模式"（时间流速×2）
  - 12个结局：解锁"真实模式"（丧尸更强，但掉落更好）
- **新周目专属事件**：第二周目起出现的特殊对话和事件
- **跨周目成就**：全结局达成、全成就达成等

**涉及文件**：`state.js`（继承逻辑）、`save.js`（跨周目数据）、`endings.js`（结算逻辑）

## 四、续作方向建议

> 如果有人愿意做续作，以下是基于当前世界观的可扩展方向。

### 4.1 《丧尸世界2：重建》

**时间线**：新势力结局后10年

**核心玩法**：从生存转向重建——建设城镇、管理人口、制定法律、处理派系冲突

**关键设定**：
- 改良血清已量产，但并非所有丧尸都能被治愈
- "清醒丧尸"群体争取平等权利，引发社会分裂
- 新的变异体出现——病毒在10年间继续演化
- 玩家是"新势力"的领导者，面对的不是生存问题，而是治理问题

### 4.2 《丧尸世界：岛》

**时间线**：与原作平行

**核心玩法**：岛屿生存——资源有限、四面环海、丧尸从海中登陆

**关键设定**：
- 玩家是岛屿上的普通居民，不是余墨公爵
- 从底层视角体验岛屿生活——城堡的压迫、余墨的挣扎、岛民的日常
- 海洋是新的探索维度——潜水、航行、海底废墟
- 最终选择：继续效忠城堡、跟随余墨独立、或自己造船离开

### 4.3 《丧尸世界：零》

**时间线**：病毒爆发前3天 → 爆发当天

**核心玩法**：倒计时叙事——你知道末日要来，但你能做什么？

**关键设定**：
- 玩家是"重生"项目的前成员
- 3天时间内：警告同事、销毁资料、逃离实验室、或试图阻止
- 每个选择都影响末日的规模
- 多结局：阻止成功（小规模泄漏）、阻止失败（原作时间线）、加速泄漏（更糟的时间线）

## 五、技术债务与优化方向

> 如果只是做优化而不加新内容，以下是值得处理的点。

### 5.1 代码层面

| 优先级 | 问题 | 建议 |
|--------|------|------|
| 高 | `maps.js` 近2000行 | 拆分为 `map-events/` 子目录，每个地图一个文件 |
| 高 | 部分模块循环依赖 | 引入事件总线或依赖注入解耦 |
| 中 | 对话文本硬编码在逻辑文件中 | 对话文本应全部移至 `dialogues/` 目录 |
| 中 | 存档格式无版本号 | 添加 `saveVersion` 字段，方便未来迁移 |
| 低 | 缺少单元测试 | 至少为 `state.js`、`combat.js` 添加核心逻辑测试 |

### 5.2 性能层面

| 优先级 | 问题 | 建议 |
|--------|------|------|
| 中 | 大量DOM操作无虚拟DOM | 考虑引入轻量级响应式更新机制 |
| 低 | 图片/音频资源未预加载 | 如果未来添加多媒体内容，需要预加载策略 |
| 低 | 无Service Worker | 添加SW实现离线可用 |

### 5.3 体验层面

| 优先级 | 问题 | 建议 |
|--------|------|------|
| 高 | 无新手引导 | 添加可选的引导流程（前3天） |
| 中 | 战斗日志不够直观 | 添加战斗动画/图标反馈 |
| 中 | 物品管理繁琐 | 添加批量使用/批量丢弃功能 |
| 低 | 无音效 | 添加环境音效和战斗音效（可关闭） |
